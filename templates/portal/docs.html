How DALA Analytics Works

A plain-language guide to everything in the system — no technical knowledge needed.

Version 3.0 Updated March 2026

What is DALA Analytics?

The simple explanation

Every month, DALA Technologies exports sales data from Tally (the accounting software) as an Excel file. That file contains every sale, every inventory movement, and every brand partner's numbers — all in one place.

DALA Analytics takes that Excel file and automatically:

  • Calculates performance numbers (revenue, stores reached, repeat orders, stock levels) for all 28+ brand partners
  • Generates a professional PDF report and an interactive web report for each brand
  • Saves everything to a database so you can compare months side-by-side
  • Sends reports directly to brand partners by email or WhatsApp
  • Shows forecasts, rankings, and alerts so you know what needs attention

Think of it as your automatic monthly reporting team — upload one file, get 28+ reports in minutes.

Step 1 — Export from Tally

In Tally, export your sales ledger as an Excel (.xls or .xlsx) file covering the full period you want. The system also accepts .csv files for large historical datasets.

Step 2 — Open the Import page

Click "Import Data" in the left sidebar. This opens the 6-step Import Wizard.

Step 3 — Choose dates and upload

Use the Quick Select buttons (This Week / Last Week / This Month / Last Month / Last 3 Months) to fill in the dates automatically, or type them manually. Then drag your file onto the upload zone and click "Analyze File". The system profiles your data in under 3 seconds — no reports yet, just a quick preview.

Step 4 — Review what was found

You'll see brand revenue bars, a date calendar (which days had sales), a data quality score, and a comparison to your last report. This lets you spot any problems before committing.

Step 5 — Configure and Generate

Choose which brands to include (default: all of them). Click "Generate Reports". A live progress bar shows each brand being processed. When done, links to all outputs appear.

What gets created: One PDF per brand  ·  One interactive web report per brand  ·  A Portfolio Dashboard covering all brands  ·  All numbers saved to the database

Weekly vs Monthly vs Quarterly Reports

Understanding report period types — how separation works

You can generate reports for any date range — a single week, a full month, a quarter, or something custom. The system automatically detects and labels the type based on the number of days you selected. All report types are stored together in the same database — they don't conflict with each other.

TypeHow many daysLabel exampleBest for
Weekly 1 – 7 days "Week of 03 Feb 2026" Checking a promo week or tracking a specific event
Biweekly 8 – 14 days "03 Feb – 16 Feb 2026" Mid-month check-ins
Monthly 28 – 31 days "Feb 2026" Standard monthly reports — the most common use case
Quarterly 85 – 95 days "Q1 2026" Quarterly reviews and board presentations
Custom Any other range Start date as label Special analysis windows

On the Import page, use the Quick Select buttons to set dates instantly:

  • This Week — Monday to Sunday of the current week
  • Last Week — The previous full Monday–Sunday week
  • This Month — 1st to last day of the current month
  • Last Month — Full previous calendar month
  • Last 3 Months — A quarterly window
  • Custom Range — You set both dates manually
Can I run a weekly AND a monthly report for the same month? Yes — they're stored separately and won't overwrite each other. Weekly reports let you track mid-month progress before the full month's report is ready.

What the Numbers Mean

Every KPI explained in plain English

Revenue and Sales

  • Total Revenue (₦) — Total naira value of all goods sold by this brand in the period.
  • Total Quantity — Number of units sold (cases, packs, tins, etc.).
  • WoW Change — Week-over-Week change. If last week was ₦200K and this week ₦250K, WoW is +25%.

Store Performance

  • Stores Reached — How many different supermarkets/outlets bought this brand's products.
  • Repeat Rate (%) — What percentage of stores placed more than one order. High repeat rate means stores are restocking, not just trying the brand once. This is one of the most important health signals.
  • Revenue per Store (₦) — Average revenue from each store relationship.
  • Top Store — The single outlet that generated the most revenue for this brand.

Products (SKUs)

  • Unique SKUs — How many different products this brand sold. SKU = Stock Keeping Unit, meaning each distinct product name.

Inventory

  • Closing Stock — Units left in stock at the end of the period.
  • Stock Days Cover — At the current sales rate, how many days will this stock last? Under 7 days = critical (urgent restock needed). Under 14 days = warning.
  • Inventory Health — Healthy / Watch / Low Stock — a simple label based on days cover.

Performance Grade (A / B / C / D)

A single overall grade built from four components:

  • Revenue Score (40%) — How this brand's revenue compares to the portfolio average
  • Loyalty Score (25%) — Repeat purchase rate
  • Reach Score (20%) — Number of stores compared to top performer
  • Activity Score (15%) — How many trading days in the period

A (80–100) = Excellent  ·  B (60–79) = Good  ·  C (40–59) = Average  ·  D (0–39) = Needs attention

Alerts — What Gets Flagged Automatically

8 things checked after every import
  • Revenue below portfolio average — If a brand earns less than 50% of the average, high-priority alert.
  • Month-on-month revenue drop — Drop over 25% = high alert. Drop 10–25% = medium alert.
  • Low stock — Stock days cover below 7 = critical. Brand needs restocking urgently.
  • No repeat orders — If no store reordered in the entire period, there's a loyalty problem.
  • Single-SKU risk — One product responsible for 90%+ of revenue = dangerous concentration.
  • Inactive brand — A previously active brand with zero sales this period.
  • Revenue milestones — Positive alerts when a brand first crosses ₦5M, ₦10M, or ₦20M.
  • Portfolio concentration — One brand making up 40%+ of total portfolio revenue is a business risk.

You can add your own rules on the Alert Rules page. Once you've handled an alert, click "Acknowledge" to dismiss it.

AI-Powered Brand Summaries

What the AI does — and what it writes for you

After every report is generated, DALA Analytics can automatically write a professional performance summary for each brand partner. These are real sentences — not tables — written by Google Gemini AI, trained on the brand's actual numbers.

What gets written?

Each brand gets a 3–4 sentence paragraph that covers:

  • How revenue compares to last month and the portfolio average
  • Key strengths (e.g. high repeat rate, growing store reach)
  • One area of concern (e.g. low stock cover, declining trend)
  • A clear recommendation for the DALA team
Example output: "Orisirisi delivered a strong February performance with ₦1.3M in revenue across 14 stores, up 22% month-over-month. Repeat purchase rate of 47% is the highest in the portfolio, indicating excellent store loyalty. However, stock cover has dropped to just 6 days — urgent restocking is recommended before mid-March to avoid a sales gap."

How to use it

  1. Add your GEMINI_API_KEY to Railway Variables (get one free at aistudio.google.com)
  2. Import a report as normal — narratives generate automatically in the background
  3. View narratives on each Brand Detail page or in the brand partner PDF
  4. To regenerate: visit /api/narrative/[Brand Name]?regenerate=1
The AI also writes an executive portfolio summary — a single board-ready paragraph covering the entire month's performance across all brands. Access it at /api/narrative/portfolio.

WhatsApp Bot

Get live reports by sending a WhatsApp message

Anyone on the DALA team can WhatsApp the DALA number and instantly get a report — no need to open a browser or log in.

Available commands

  • REPORT — Full portfolio summary (total revenue, brands, top performer)
  • REPORT Orisirisi — Summary for a specific brand (partial names work)
  • ALERTS — List of all current unacknowledged alerts
  • BRANDS — Full list of all active brand partners
  • HELP — Show the command menu
The bot also includes the AI-generated summary for that brand (if AI is configured) at the bottom of the message.

How to set up

  1. Go to your Twilio account → WhatsApp Sandbox (or Business account)
  2. Set the "When a message comes in" webhook URL to:
    https://your-railway-domain.railway.app/webhook/whatsapp
  3. Make sure TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, and TWILIO_WHATSAPP_FROM are in your Railway Variables
  4. Save — messages sent to your Twilio WhatsApp number will now be answered automatically

Google Drive Auto-Sync

Build the complete historical database from your Drive folders — automatically

DALA watches two Google Drive folders (2025 and 2026 Sales Reports) and can import every Excel file into the database with one click, building the full multi-year history that powers Forecasting, Trends, and the Leaderboard.

Setting It Up

  1. Run python setup_oauth.py on your local machine — this opens a Google sign-in page and saves google_token.json to the project root.
  2. Commit google_token.json to Git (or add it to Railway as a file/env variable).
  3. The same OAuth2 token is shared with Google Sheets — if Sheets sync already works, Drive Sync works too.

Using Drive Sync

  • Import All History — Downloads every Excel file from both Drive folders and runs the full pipeline (KPIs, alerts, scorecards). Use this once to build the database from scratch. Takes ~1–2 minutes per monthly file.
  • Check for New Files — Only downloads files that are new or have changed since the last import. Use this each month after dropping the new report in Drive.
  • Each file's import status (Pending / Imported / Error) is tracked in the database and survives server restarts.

Drive Sync API

  • POST /api/drive-sync/trigger — Check for new/changed files only
  • POST /api/drive-sync/full-import — Start full historical import (async background job)
  • GET /api/drive-sync/job/[job_id] — Poll job progress (returns progress %, current file, status)
Note: Drive Sync builds the KPI database only (no PDFs/HTMLs). Brand PDFs are generated on-demand from the Brand Partners page or by re-running generation from the Home page for any specific date range.
