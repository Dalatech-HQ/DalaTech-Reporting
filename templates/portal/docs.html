🏗️ Architecture Overview

DALA Analytics is a Flask-based web application that processes Tally ERP export data to generate sales reports, insights, and forecasts for brand partners.

Key Components

  • Flask Web Server: Handles HTTP requests, serves templates and static files
  • SQLite Database: Stores reports, KPIs, alerts, and brand tokens
  • Pandas: Data manipulation and analysis engine
  • Playwright: PDF generation from HTML templates
  • Chart.js: Interactive visualizations in the browser

📊 Data Flow

1. File Upload (Ingestion)

When a Tally Excel file is uploaded:

  1. load_and_clean() in modules/ingestion.py reads the file
  2. Column names are standardized (e.g., 'Brand Partners' → 'Brand Partner')
  3. Data types are converted (dates, numeric values)
  4. Invalid rows are filtered out
Key File: modules/ingestion.py — Modify this to change how data is parsed.

2. KPI Calculation

For each brand, the system calculates:

  • Total revenue and quantity sold
  • Unique stores and SKU count
  • Repeat purchase rate (% of stores with multiple orders)
  • Stock levels and days of cover
  • Week-over-week growth rates
  • Performance scores (0-100 grading system)
Key File: modules/kpi.py — Add new metrics here.

3. Historical Analysis

The modules/historical.py module provides:

  • Month-over-Month (MoM) growth calculations
  • Trend detection (accelerating/decelerating/stable)
  • Insights generation ("What's working / not working")
  • Color scheme rotation by month

💾 Database Schema

The system uses SQLite with the following tables:

reports

id | month_label | start_date | end_date | total_revenue | brand_count | generated_at

brand_kpis

id | report_id | brand_name | total_revenue | num_stores | repeat_pct | perf_grade | ...

daily_sales

id | report_id | brand_name | date | revenue | qty

alerts

id | report_id | brand_name | alert_type | severity | message | acknowledged

brand_tokens

id | brand_name | token | email | whatsapp | active
Location: dala_data.db in project root. Use any SQLite browser to inspect.

📦 Module Guide

modules/ingestion.py

Data loading and cleaning functions:

  • load_and_clean(file) — Main entry point for file processing
  • filter_by_date(df, start, end) — Date range filtering
  • split_by_brand(df) — Separate data by brand partner

modules/kpi.py

Metric calculations:

  • calculate_kpis(brand_df) — Full KPI suite for a brand
  • calculate_perf_score(kpis, portfolio_avg) — 0-100 scoring
  • generate_narrative(kpis) — Human-readable summary

modules/historical.py

Trend analysis (NEW):

  • get_brand_monthly_history(df, brand) — Monthly metrics with MoM
  • generate_insights(history) — AI-style recommendations
  • get_color_scheme_for_month(y, m) — Monthly color rotation

modules/charts_html.py

Chart generation for PDF reports:

  • 14 chart types: top stores, products, trends, heatmaps, sparklines
  • Returns base64-encoded PNG images

modules/geocoding.py

Store location mapping (NEW):

  • geocode_store(name) — Convert store name to lat/lng
  • Caching system to minimize API calls

➕ Adding New Features

Step 1: Add Backend Logic

Create or modify modules in modules/:

  1. Create your function (e.g., modules/new_feature.py)
  2. Add unit tests if applicable
  3. Import in app.py

Step 2: Create API Route

Add to app.py:

@app.route('/api/new-feature')
def new_feature():
    result = your_function()
    return jsonify(result)

Step 3: Build Frontend

Create template in templates/portal/:

  1. Extend base.html
  2. Add navigation link in sidebar
  3. Use Chart.js for visualizations

Step 4: Update Documentation

REQUIRED: Update this page (templates/portal/docs.html) with your changes!
Also update the version badge at the top.

Google Sheets Integration

To enable Google Sheets export:

  1. Run python setup_oauth.py to authenticate
  2. Or place service account JSON at google_credentials.json
  3. Set folder ID in google_folder_id.txt
