What is DALA Analytics?

DALA Analytics is a full-stack web application built for DALA Technologies to process monthly sales data exported from Tally ERP. It automatically generates brand-level performance reports, tracks historical trends, forecasts future revenue, and delivers insights directly to brand partners — all from a single Tally Excel upload.

All Pages & Features

Home ( / )

The home page shows the latest report summary, quick-access upload card, recent report history, and system health. It's your starting point for every new month's analysis.

Import Data ( /import )

A 6-step intelligent wizard that guides you from raw file to full report set:

  1. Upload — Drag-drop a Tally .xlsx or .csv file. Select the date range you want to analyse.
  2. Analyzing — The file is instantly profiled in under 3 seconds (no reports generated yet). You see brand count, row count, date coverage, and quality issues.
  3. Review — Full preview: brand revenue bars, date calendar heatmap, top stores, top products, data quality score (0–100), missing-brand warnings, vs-last-month comparison.
  4. Configure — Choose exactly which brands to include. Override dates if needed.
  5. Generating — Live per-brand progress bars while PDFs, HTMLs, and database records are built in the background. Page polls every 600ms.
  6. Done — Direct links to Dashboard, Brand list, Portfolio HTML, and Alerts.
Large files: CSV files are read in 50,000-row chunks so even multi-year datasets (hundreds of MB) are processed without running out of memory. Excel files up to ~2 GB are supported.

Trends ( /trends )

Historical trend analysis across all months of imported data (currently 21 months: May 2024 – Feb 2026).

  • Month-over-Month (MoM) revenue growth with colour-coded change indicators
  • Monthly colour theme rotation for visual variety
  • "What is working / not working / next steps" AI-style insight bullets
  • Store repeat purchase analysis with interactive Google Maps
  • Top stores by revenue with regional clustering

Dashboard ( /dashboard )

Portfolio-level KPI overview for the most recently generated report. Shows total revenue, brand count, alert strip, and brand ranking table. All numbers come from the SQLite database — no file reading on load.

Brand Partners ( /brands )

Searchable table of all brands with performance grade badges, PDF/HTML download links, portal token sharing, and email/WhatsApp delivery buttons.

Brand Detail ( /brand/<name> )

Deep-dive for a single brand: KPI cards, 12-month Chart.js revenue history, forecast + stock depletion date, performance scorecard bars, and a one-click "Sync to Google Sheets" button.

Forecasting ( /forecasting ) — NEW in v3.0

Uses modules/predictor.py to project next month's revenue for every brand using a linear least-squares trend fitted to historical data.

  • Growing / Stable / Declining classification per brand
  • Confidence level: High (≥6 months, R² ≥ 0.7), Medium, Low
  • Stock depletion date with urgency colour (critical / warning / ok)
  • Portfolio bar chart showing all 28 brands' forecasts side by side

Brand Leaderboard ( /leaderboard ) — NEW in v3.0

Gamified brand rankings with Gold/Silver/Bronze podium for top 3 revenue brands. Four switchable categories: Revenue, Store Reach, Repeat Rate, Performance Score. Inline progress bars show each brand's relative standing.

SKU Analytics ( /sku-analytics ) — NEW in v3.0

Product-level analysis. Top 15 SKUs by revenue (horizontal bar chart), SKU count by brand (doughnut chart), and a searchable/filterable table covering every SKU in the current report. One-click CSV export.

Targets ( /targets ) — NEW in v3.0

Set monthly revenue targets per brand. Each brand card shows a circular progress ring (green ≥ 80%, amber 50–79%, red < 50%). Summary cards show On Track / At Risk / Off Track / No Target counts portfolio-wide. Targets survive across imports and are stored in the brand_targets SQLite table.

History ( /history )

Month selector with Chart.js bar chart showing portfolio revenue across all imported months. Click any month to drill into that report.

Compare ( /compare )

Side-by-side comparison of any two brands from the same report. KPI cards, performance scores, and a VS divider layout.

Alerts ( /alerts )

Smart alert dashboard. Alerts are auto-generated on every import by modules/alerts.py (8 built-in rules). Unacknowledged count shown as red badge on nav and topbar bell. Click "Acknowledge" to dismiss.

Alert Rules ( /alert-rules ) — NEW in v3.0

Custom alert rule builder. Create rules that fire on any metric (revenue, repeat %, stock days, WoW change, performance score) with any comparison operator and threshold. Rules can be paused, resumed, or deleted. 5 pre-built suggestion templates included.

Settings ( /settings )

SMTP email config display, Twilio WhatsApp config, and brand token management table where you can update email/WhatsApp contacts per brand, regenerate access tokens, and copy portal links.

Brand Portal ( /portal/<token> )

Standalone brand-facing view (no sidebar). Each brand gets a unique UUID token link. Shows their KPIs, performance grade circle, revenue forecast, 12-month history chart, and PDF download button. Tokens can be regenerated from Settings.

How It Works ( /how-it-works )

This page. Always updated to reflect the current state of the system.

Data Pipeline

1. File Ingestion ( modules/ingestion.py )

Accepts .xlsx, .xls (Tally OLE2 format), and .csv files.

  • Excel: tries openpyxl first, falls back to xlrd for binary .xls
  • CSV: reads in 50,000-row chunks (chunksize=50000) — no RAM ceiling
  • Column renames applied: Brand Partners → Brand Partner, Retailers → SKUs, Value → Sales_Value
  • Type casting: dates parsed, quantities / values converted to float, strings stripped
  • Rows with unparseable dates are dropped

2. KPI Calculation ( modules/kpi.py )

Per-brand metrics computed from the date-filtered DataFrame:

KPIDescription
total_revenue / total_qtySum of Sales rows
num_storesUnique Particulars (supermarkets/outlets)
unique_skusUnique Retailers (product names)
repeat_pct% stores with 2+ distinct Vch No. (reorders)
stock_days_coverClosing stock ÷ avg daily sales
perf_score / perf_gradeComposite 0–100 score → A/B/C/D
wow_rev_changeWeek-over-week revenue % change
peak_date / peak_revenueHighest single-day revenue

3. Database Storage ( modules/data_store.py )

SQLite with WAL journaling. Six tables:

TableContents
reportsOne row per import run (dates, totals, brand count)
brand_kpisFull KPI set per brand per report
daily_salesDaily revenue + qty per brand per report
alertsAuto-generated alerts from each import
brand_tokensUUID portal tokens + contact details per brand
brand_targetsMonthly revenue targets set manually
alert_rulesCustom threshold rules for auto-alerts
activity_logAudit trail: imports, syncs, target changes

4. Alert Generation ( modules/alerts.py )

8 built-in rules checked after every import:

  • Revenue vs portfolio average (below 50% → high alert)
  • MoM revenue drop > 25% (high) or > 10% (medium)
  • Low stock: closing stock < 7 days cover
  • No reorders in the period
  • Single-SKU risk (only 1 product generating all revenue)
  • Brand inactive (no sales rows in date range)
  • Revenue milestones: ₦5M, ₦10M, ₦20M crossed
  • Portfolio concentration risk (top brand > 40% of total)

Custom rules from the Alert Rules page layer on top of these built-in rules.

5. Forecasting ( modules/predictor.py )

Linear least-squares trend fitted to all available monthly revenue history.

  • Forecast = last known revenue + slope
  • R² determines confidence: High (≥ 0.7, ≥ 6 months), Medium (≥ 0.4, ≥ 3 months), Low otherwise
  • Stock depletion date = report end date + stock_days_cover
  • Growth label: Growing (+5%/mo slope), Declining (−5%), Stable

6. Report Generation

Two output formats per brand:

  • PDF — Generated by Playwright (Chromium) rendering an HTML template. Contains charts, narrative, KPI tables. Stored in output/pdf/.
  • Interactive HTML — Standalone Plotly-powered report. No server needed to open. Stored in output/html/.
  • Portfolio HTML — Single file covering all brands with a brand selector dropdown (client-side, zero server calls). Stored in output/html/.

Google Integrations

Google Maps ( modules/geocoding.py )

Store names from the Particulars column (e.g., "Jendol Supermarket, Ajah") are geocoded via the Google Maps Geocoding API to latitude/longitude coordinates. These power the interactive store map in the Trends page.

  • Requires GOOGLE_MAPS_API_KEY environment variable
  • Results are cached to minimise API calls
  • Map shows bubble size = store revenue, colour = revenue intensity

Google Sheets ( modules/sheets.py )

Each brand's data can be pushed to a branded Google Sheet with DALA formatting (navy header, frozen row, alternating rows, view-only link sharing).

  • Auth: uses google_credentials.json (service account) or google_token.json (OAuth2 via python setup_oauth.py)
  • Trigger from Brand Detail page → "Sync to Sheets" button, or via POST /api/sync_sheets/<brand>
  • Returns a shareable URL embeddable in PDFs

Report Delivery ( modules/delivery.py )

Email

PDF reports sent via SMTP (Gmail App Password recommended). Each brand's PDF is attached. Configure in environment variables or Settings page.

SMTP_HOST=smtp.gmail.com  SMTP_PORT=587
SMTP_USER=your@gmail.com  SMTP_PASSWORD=app-password

WhatsApp

Summary messages sent via Twilio WhatsApp API. The message includes brand name, revenue, top store, repeat rate, and grade. Requires a Twilio WhatsApp-enabled number.

TWILIO_ACCOUNT_SID=ACxx...  TWILIO_AUTH_TOKEN=...
TWILIO_WHATSAPP_FROM=whatsapp:+14155238886
Both email and WhatsApp contact details per brand are set in Settings → Brand Token Management.

REST API v1 — NEW in v3.0

Public JSON endpoints for integrations (Tally API, Power BI, external dashboards).

GET/api/v1/brands — All brands with latest KPIs GET/api/v1/kpis/<brand_name> — 24-month KPI history for one brand GET/api/v1/alerts — All unacknowledged alerts (JSON) GET/api/v1/portfolio — Portfolio monthly revenue trend GET/api/export/brands?format=csv — Download brand KPIs as CSV GET/api/export/skus?format=csv — Download SKU summary as CSV GET/api/export/alerts — Download all alerts as CSV POST/api/sync_sheets/<brand> — Push brand data to Google Sheets
All endpoints return JSON unless a CSV format is specified. No authentication is required for GET endpoints (add Railway basic auth if needed in production).

Google Credentials Setup

  1. Go to Google Cloud Console
  2. Create or open project dala-report-automation
  3. Enable: Sheets API, Drive API, Maps JavaScript API, Geocoding API
  4. Service Account JSON already at google_credentials.json — do not commit
  5. For Drive file ownership: run python setup_oauth.py once to create google_token.json
  6. Set target Google Drive folder in google_folder_id.txt

Railway Deployment

The app deploys to Railway via nixpacks.toml which installs Chromium + all X11 libs for Playwright PDF generation. The gunicorn start command is set to 2 workers with a 120s timeout.

Push to main branch on GitHub → Railway auto-deploys in ~3 minutes.
